Tratamento de erros e exceções

Try/Catch/Finally

As declarações try...catch marcam um bloco de declarações para testar (try), e especificam uma resposta, caso uma exceção seja lançada.

  • try_statements
    • Declarações a serem executadas.
  • catch_statements
    • Declarações a serem executadas se uma exceção for lançada no bloco try.
    O bloco catch especifica um identificador que contém o valor especificado pela throw instrução.
    Esse identificador pode ser usado para obter informações sobre a exceção lançada. JavaScript cria este identificador quando o bloco catch é inserido. O identificador dura apenas a duração do bloco catch. Uma vez que o bloco catch termina de ser executado, o identificador não existe mais.
    NOTA: É recomendado usar console.error() em vez de console.log() para depuração. Ele formata a mensagem como um erro e adiciona à lista de mensagens de erro geradas pela página.
  • finally_statements
    • Declarações a serem executadas depois que a declaração try é completada.
    Essas declarações são executadas independente se uma exceção foi ou não lançada ou capturada.
    Se o bloco finally retornar um valor, esse valor se tornará o valor de retorno de toda a try…catch…finally, independente de qualquer return declarado nos blocos try...catch. 

                            try {
                                // código que pode gerar erro
                            } catch (error) {
                                // tratamento do erro
                            } finally {
                                // código que sempre será executado (Se houver erro ou não)
                            }
