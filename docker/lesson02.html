Docker

Terminologias

  • Container Image: Pacote de dependências para criação de novos containers
    • Uma imagem do Docker é um sistema de arquivos privado apenas para seu contêiner.
    Ele fornece todos os arquivos e códigos que seu contêiner precisa.
  • Dockerfile: Arquivo de texto contendo todas as instruções para fazer build da imagem
  • Build: Ação que cria uma imagem a partir de um Dockerfile
  • Container: Instância de Container Image, representa a execução de uma app, processo ou serviço
  • Volumes: Dados em disco armazenados pelo container, se este morrer, dados permanecem
  • Tag: Ajuda no versionamento das imagens
  • Multi-stage build: Multiplos estágios de build
  • Repository: É uma coleção de imagens
  • Registry: Serviço que provê acesso do docker ao repositório
  • Docker Hub: Repositório de imagens públicas e privadas
  • Compose: Ferramenta que usa metadados para criar múltiplos containeres com um simples comando

O que é

Plataforma de código aberto de alto desempenho, que permite criar e administrar ambientes de forma isolada.
O objetivo é isolar o software de forma que seja possível instalar em diferentes ambientes, permitindo assim, criar, testar, implementar em um ambiente separado do ambiente de produção. Também permiti disponibilizar de forma rápida para o usuário final.
Esse processo é chamado de conteinerização.

Ferramenta para construir, compartilhar e executar aplicativos com segurança e rapidez.

O Docker é um conjunto de produtos de plataforma como serviço que usam virtualização no nível do sistema operacional para entregar software em pacotes chamados contêineres. Os contêineres são isolados uns dos outros e agrupam seus próprios softwares, bibliotecas e arquivos de configuração; eles podem se comunicar uns com os outros por meio de canais bem definidos.

Como funciona


Exemplo: O Docker Client a partir de um arquivo Dockerfile pode executar um build para criar uma imagem e a partir desta executar um docker run para criar um container.
Também podemos executar um docker pull para puxar a imagem do repositório remoto para dentro do repositório local e executar um docker run para criar um container a partir dessa imagem.

Docker Hub

hub.docker.com
Possui uma nuvem de containers e imagens pré-configurados.
Existem as oficiais e as que os desenvolvedores criam e compartilham.

Docker Compose

É um arquivo que especifica as propriedades de cada container. Permiti fazer links entre diferentes containers, para que exista todo um ecossistema do software configurado, e assim subir o software com todas as suas dependências.
É uma receita passo a passo, com todo o ambiente simplificado, para isso vai usar o dockerfile.

Play With Docker

É um playground do Docker que permite aos usuários executar comandos do Docker em questão de segundos.
Oferece a experiência de ter uma máquina virtual Linux Alpine gratuita no navegador, onde você pode construir e executar contêineres do Docker e até mesmo criar clusters no modo Docker Swarm. Sob o capô Docker-in-Docker (DinD) é usado para dar o efeito de várias VMs/PCs.

Inclui um site de treinamento composto por um grande conjunto de laboratórios e questionários do Docker, do nível iniciante ao avançado, disponíveis em training.play-with-docker.com.

Uma instância seria um servidor físico com Docker instalado, também poderia ser uma VM com o docker instalado.
Comandos no terminal: 

                            // Lista os containers e seus status
                            docker stats

                            // Faz um pull da imagem e criar um container
                            docker run --name my-first-container -p 80:80 -d nginx

                            // Lista as imagens
                            docker images

Docker Desktop

Após a instalação, abrir o power shell e executar o comando para verificar: 

                            docker --version
                            docker ps
                            docker run hello-world
O Docker Desktop cria uma máquina virtual (VM) dentro do virtualizador Hyper-V do Windows.
OBS: Use Docker Desktop somente para ambiente de desenvolvimento.
Para ambiente de produção use a nuvem com linux.
