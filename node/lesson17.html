Microserviço para Autenticação e Gerenciamento de Usuários - REST API

Testar o Microserviço

Obter um token de autorização

É como o login da app.
  • Executar endpoint http://localhost:3000/token
  • Método HTTP POST
  • Forma de autenticação Basic Auth
  • Informar as credenciais do usuário (userName e password)
O Token retornado aqui será obrigatório para todos os endpoints do CRUD de usuários.

Validar um Token

  • Executar o endpoint http://localhost:3000/token/validate
  • Método HTTP POST
  • Forma de autenticação Bearer Auth
  • Informar o token

Testar o CRUD de usuários

  • Executar os endpoints, por exemplo: http://localhost:3000/users
  • Forma de autenticação Bearer Auth
  • Informar o token
Todos os endpoints do CRUD de usuários estão protegidos por um token de autorização.
Isso porque a rota de usuários usersRoute está condigurado no server em src/index.ts
O middleware é o responsável por validar o token de autorização. 

                            // ANTES sem token
                            app.use(usersRoute);

                            // DEPOIS com token
                            app.use(bearerAuthentication, usersRoute);
