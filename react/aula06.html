Renderização

A renderização é o processo de transformar um componente em HTML, e é realizada através do método render() do objeto ReactDOM.
O elemento é a menor unidade de renderização, e toda a renderização é feita em um nó raíz (Não é possível retornar nós irmãos).
Um elemento NÃO é um componente. Podemos dizer que um componente é um conjunto de elementos.
Um elemento pode ser uma string, um html, ...
OBS: Toda renderização será realizada em public/index.html, está será a única página da app. Conceito de SPA - Single Page Application. React "pensa" em como a interface deve estar a cada momento ao invés de alterá-la com o tempo, o que faz ter ganho de performance e evita uma séria de erros.

Virtual DOM

É uma cópia do DOM (Document Object Model).
O browser tem o DOM que é uma representação de um documento.
O React não trabalha diretamento com o DOM do browser, mas sim com o Virtual DOM.
Por exemplo, quando temos uma alteração em algum elemento, o ReactDOM não vai renderizar todo o objeto (toda a estrutura) novamente.
O que ele vai fazer é realizar uma diferença, uma comparação entre o Virtual DOM e o DOM do browser, e assim ele renderiza novamente somente aquele pedaço que foi modificado.
