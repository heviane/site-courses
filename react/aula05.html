Curso de React.js

O que é JSX?

JSX apenas fornece syntactic sugar para a função React.createElement do React.
É uma linguagem de marcação que permite misturar html e javascript.
Não é HTML, Não é JavaScript. É uma mistura dos dois.
O código JavaScript deve ser colocado dentro de um par de chaves {}.
O browser não interpreta JSX, para isso é necessário um transpilador de código. O transpilador mais utilizado é o Babel.

Por que usar JSX?

A utilização do JSX não é obrigatória. Mas, se não usado, será necessário chamar a função React.createElement para criar um elemento, e passar todos parâmetros necessários. Sendo assim, JSX torna muito mais simples a renderização do código. O compilador Babel online pode ser usado para transpilar (converter) o código JSX para o código JavaScript.

The JSX code:

                            const element = Hello World!
Compiles into:

                            const element = React.createElement(
                                'h1',
                                {className: 'title'},
                                'Hello World!'
                            );

React Component

A primeira parte da tag JSX é o nome do elemento React CustomButton.
Este deve iniciar com letra maiuscula e deve ser importado, pois essas tags são compiladas em uma referência direta a variável noemada, portanto deve estar no escopo através da importação.
Como o JSX compila em chamadas para React.createElement, a biblioteca React também deve estar sempre no escopo do seu código JSX.

Portanto, ambas as importações são necessárias neste código, mesmo que React e CustomButton não sejam referenciados diretamente do JavaScript. 


                        import React from 'react';
                        import CustomButton from './CustomButton';

                        function WarningButton() {
                            // return React.createElement(CustomButton, {color: 'red'}, null);
                            return <CustomButton color="red" />;
                        }
Se não usar um bundler JavaScript e carregou o React de uma tag <script>, ele já está no escopo como o React global.

Notação de ponto para o tipo JSX

Outra forma de fazer referência a um componente React é usando notação de ponto de dentro do JSX. Isso é conveniente se você tiver um único módulo que exporte muitos componentes React.

Exemplo: Se MyComponents.DatePicker for um componente, você poderá usá-lo diretamente do JSX com: 

                            import React from 'react';

                            const MyComponents = {
                                DatePicker: function DatePicker(props) {
                                    return <div>Imagine a {props.color} datepicker here.</div>;
                                }
                            };

                            function BlueDatePicker() {
                                return <MyComponents.DatePicker color="blue" />;
                            };

Componentes definidos pelo usuário devem ser capitalizados

Quando um elemento começa com letra minúscula, refere-se a um componente interno como <div> ou <span> e resulta em uma string 'div' ou 'span' passada para React.createElement.

Quando um elemento começa com letra maiúscula compila para React.createElement() e corresponde a um componente definido ou importado em seu arquivo JavaScript.

Recomendação: Nomear os componentes com uma letra maiúscula. Se tiver um componente que comece com uma letra minúscula, atribua-o a uma variável em maiúscula antes de usá-lo no JSX.
