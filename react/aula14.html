Rest HTTP

As APIs HTTP servem para conectar a um ou mais servidores HTTP.

    Principais Métodos HTTP

  • GET: Para listagem de dados.
  • POST: Para criação de dados.
  • DELETE: Para remoção de dados.
  • PUT: Para modificações totais de dados.
  • PATCH: Para modificações parciais de dados.

    Abordagens

  • Fetch API Library
    • API nativa do browser com interface JavaScript.
    É recente, alguns browsers ainda não suportam. Opção usar Node Fetch.
    Suporte a Service Workers (Foi por isso que surgiu).
    Oferece uma alternativa ao XMLHttpRequest() e JQuery.ajax() com algumas diferenças:
    - Não envia nem recebe cookies (precisa definir a opção credentials).
    - Não rejeita o status do erro HTTP (só rejeita em caso de instabilidade de conexão).

  • Axios Library
    • Muito usada em React.
    Axios é um cliente HTTP baseado em promessa simples para o navegador e node.js.
    Simples de usar em um pequeno pacote com uma interface bastante extensível.
    Cross-Browser (Baseada no XMLHttpRequest() do JavaScript).
    Pode monitorar o progresso de uma requisição HTTP.
    Melhor tratamento de erros.
    Melhor teste.
    Suporte bastante limitado para PWAS (Restrito ao Fetch API) .

    Como usar? É necessário baixar e instalar, pois é uma lib de terceiros. 
    
								npm i axios
                                npm install axios --save
                                yarn add axios
