Context API

Hook useContext.
Se fosse classe, seria consumer.
Forma de passar e controlar valores em um contexto global sem a utilização de componentes intermediários.
Context API tem um consumer e provider.
Pode-se ter vários providers no mesmo componente.
React não vai renderizar tudo, somente o que estiver consumindo o provider em questão.
Deve ser usado com cautela: Pode sobrecarregar com re-renderizações.
Existem outras ferratmentas controladoras de estado.
Nem sempre vai ser necessário usar um contexto global.

Exemplo prático:

AppContextApi.js está passando os valores para o provider.
O provider vai providenciar os valores para os componentes filhos.
Theme.js é a criação do contexto inicial.
Card2.js e Form2.js são os componentes que estão consumindo o contexto.
Card2.js está estilizando o button com os valores iniciais criados em theme.
Form2.js está validando o token, que foi criado depois no contexto na AppContextApi.js
