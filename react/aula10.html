Estilização

Formas de estilização
Cada propriedade é declarada no formato camelCase.

Inline

Estilização inline através do atributo style que é um objeto que gera um json.
Opção 1: Podemos declarar um elemento em uma variável contendo todo o bloco de estilos e então passá-lo entre chaves {} para o atributo style.

                            // Declaração do elemento
                            const H1Style = {
                                color: 'blue',
                                textalign: 'center'
                            };
                            // Implementação do estilo
                            <h2> style={H1Style}> { props.children } </h2>
Opção 2: Podemos declarar a propriedade css entre chaves duplas {{}} diretamente no atributo style.

                            <h2> style={ {color: 'brown'} }> { props.children } </h2>
Vantagens:
  • Podemos implementar urls dinâmicas.

    • Classes

    Estilização com classe é uma forma mais modularizada.
    Deve ser criado um arquivo .css para realizar a declaração de todas as classes.
    Assim, basta importá-lo no componente e declarar as classes no atributo className do elemento.
    O atributo className que é um array de strings.
    OBS: Usa-se a palavra className porque a palavra "class" é reservada no React.
    Para manter a compatibilidade com element.className no jsx, e não ao "class" do DOM.
    
                            // Implementação do estilo
                            import './containers/App.css';
    Vantagens:
  • Maneira mais prática e direta
    • Desvantagens:
  • Difícil manutenção
  • Pouca flexibilidade
    • Não é possível usar uma url dinâmica.
    Na forma inline poderíamos fazer isso.
  • Conflito com nomes (mais comum)

    • CSS in JS (forma mais recente)

    O css é criado dentro do JavaScript e este gera css para a App.
    Existem diversas abordagens.
    Abordagem styled-components é a mais usada em React.
    Nesta abordagem, usa-se css puro.
    
                            npm install --save styled-components
    Vantagens:
  • Manutenção
    • Facilidade para remover css, não precisa refatorar a App inteira.
    Estilos dinâmicos usando props
  • Performance
    • Elimina necessidade de usar pré-processador de css
  • Injeção automática de prefixos vendor
    • Não precisa mais se preocupar com webkit, moz, etc.
    Ele já gera o prefixo correto e de acordo com o browser.
