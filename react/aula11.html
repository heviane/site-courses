Componentes Controlados e Componentes não controlados

DOM no HTML5 já mantém um estado interno para cada elemento do formulário.
Em React, podemos controlar o estado com state e setState.
Para fazer o merge dessde dois estados (DOM e React) há duas formas de trabalhar:

Componente controlado (Por valor)

Componente controlado é quando o estado, o Single source of truth é React.
select, input, textarea aceitam atributo value que armazena um valor.
Podemos modificar esse value usando o atributo onChange.

Componente não controlado (Por Referência)

A tag input é read-only.
Componente não controlado é quando o estado, o Single source of truth é DOM.
O React vai apenas observar e pegar uma referência do estado do elemento read-only no envio do formulário.
input type="file", no HTML5 é read-only.
Portanto, no React só podemos usar componente não controlado.
