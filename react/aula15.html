Imutabilidade

    Pilares

  • Coleção imutável (Uma vez criada não pode ser alterada)
    • Coleção seria uma tupla, array, lista, etc.
  • Persistência. Novas coleções podem ser criadas a partir de uma coleção anterior e uma mutação (setter) como um conjunto
  • Novas coleções são criadas usando o máximo possível da estrutura original, reduzindo a cópia e aumentando a performance

    Benefícios

  • Performance
    • Não estamos trabalhando com atribuiçoes, mutações, mas sim criando cópias otimizadas.
  • Programação mais simples
    • Porque estamos sempre trabalhando com estruturas fixas.
  • Debugging mais simples (detecção de mudanças)
    • Pode-se realizar shallow comparison (comparações de igualdade simples comparando valor com valor).
