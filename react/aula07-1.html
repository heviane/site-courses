Componentes

O que são componentes?

Componentização é o ponto forte do React, é o que o torna modular. É uma forma de organizar o código.
React não separa as tecnologias colocando marcação em arquivos separados, mas sim separa conceitos e mantém o código pouco acoplado chamando-os de componentes, que são pequenas peças que vão sendo acopladas ao longo do projeto para a montando de uma app, como se fosse um LEGO.
Componentes são como funções em JavaScript, aceitam entradas arbitrárias (chamadas "props") e retornam elementos que descrevem o que deve ser exibido na tela.
Permitem dividir a UI em partes independentes, reutilizáveis, encapsulando funcionalidades.
Trata cada parte como um bloco isolado, livre de dependências externas.

Por que componentizar?

  • Responsabilidade
    • Desacoplar. Cada componente deve ter uma responsabilidade única. Quanto menor a responsabilidade, melhor é.
    Preocupar-se mais com a lógica do que com conflitos de responsabilidade.
  • Facilitar a manutenção
    • Tornar a leitura, entendimento do código fonte mais clara.
  • Reutilização
    • Componente pode ser importado e reutilizado de quaiquer outros lugares.
  • Extensibilidade
    • Componente pode extender outro(s) componente(s), herdando funcionalidades.

Codificação de um componente

Um Componente também pode ser visto como um conjunto de elementos.
    Um componente pode trabalhar de duas formas
  • Classe
  • Função
    • Com a inclusão dos Hooks.
Return
O return é realizado com código JSX.
Permiti escrever código JavaScript, porém este deve ser colocado entre um par de chaves {}.
Retorna um único elemento, ou seja, um único Nó Raiz.
Para retornar um conjunto de elementos irmãos, estes devem estar devidamente encapsulados em um único elemento.
    Opções de encapsulamento:
  • Div: <div> </div>
  • Fragment: <Fragment> </Fragment>
    • O fragment tem a função de agrupar vários elementos em um único elemento a ser renderizado para o DOM/HTML.
    A vantagem disso é tornar o código final a ser enviado para o browser mais otimizado.

Boas Práticas

  • Nomeação: Usar o padrão CamelCase na criação de nomes relacionados a componentes ou páginas.
  • Significado: Usar nomes com significado, coerentes, que remetem a funcionalidade.
