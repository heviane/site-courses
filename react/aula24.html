Dumb Components

    Regras que caracterizam

  • Preocupa-se com a apresentação, a renderização
  • Recebem valores via props
  • Não possuem dependências com o restante da app
    • Ou seja, são totalmente desacoplados do restante da aplicação.
  • Não especificam como os dados são carregados ou sofrem mutações
  • Recebem valores e callbacks exclusivamente via props
  • Raramente possuem estado, quando precisam de estado é para controlar a interface e não dados do usuário
    • Não vai tratar dados, fazer mutação (alterações)
  • São escritos na maioria das vezes como componentes funcionais
    • Na maioria dos casos, são funções puras.
  • Desacoplados

    Exemplos

  • Button
    • Se tem um click, é para executar uma função externa.
  • Card
  • Sidebar
  • Footer
  • List
  • Menu

Smart Components

    Regras que caracterizam

  • Preocupam-se em como as coisas vão funcionar, comportamento
  • Podem conter Smart e Dumb Components
  • Providenciam dados e padrões de apresentação ou outros containers
  • Na maioria dos casos possuem estado e podem chamar outros fluxos do sistema
  • Possuem regras especificas, não servem para colocar em qualquer lugar
  • Acoplados

    Exemplos

  • UserGalerry
  • UserPage
  • FilterBook
  • FollowersSidebar
  • ListCards
DICA: Usar memo na exportação do componente para evitar re-render desnecessária.
Só vai renderizar novamente se houver alterações na árvore.

RESUMO

Dumb Compoenents: Desacoplados, sem comportamento (Apresentação).
Smart Components: Acoplados, com comportamento (Lógica).

É uma boa prática separar os componentes em Dumb e Smart, definir um padrão.
Melhora a manutenção, deixa código mais limpo e organizado.

Organização de Projeto

Boas práticas e recomendações para organização de um projeto em React

App.js é o main da aplicação (Rotas, providers, fluxo de autenticação, validações, tudo que for global).
Não se coloca componentes diretamente, somente rotas.
index.js é o Root Component que executa a App.js

    Estrutura de diretórios public

  • public/index.html
    • Está será a página inicial e a única página html da aplicação, é onde será realizada toda a renderização do React.
    Conceito de SPA (Single Page Application).
    Toda a navegação será feita entre componentes.

    Estrutura de diretórios src

  • src/containers
    • Todas as páginas web (UI) principais.
    • src/containers/components
      • Smart components que não são reutilizáveis.
    • src/containers/reducers
      • Se estiver usando Redux.
    • src/containers/actions
      • Se estiver usando Redux.
  • src/components
    • Todos os componentes que são reutilizáveis (Dumb e Smart Components)
    • Criar um diretório para cada componente, contendo um index.js
    • Criar um index.js para importar e exportar todos os componentes.
      Facilita para importar, basta setar o diretório principal e selecionar componentes.
      Ex: import { Button, Card, Sidebar, Footer, List, Menu } from './components';
      Simplifica a código evitando que fique muito poluído.
    • Dependendo do caso, poderia separar em Dumbs e Smarts
  • src/assets
    • Tudo que for global
    • src/assets/images
    • src/assets/style
  • src/commons
  • Tudo que for comum
    • src/commons/utils
      • Somente funções puras utilitárias e reaproveitáveis
        • Estas funções devem ter responsabilidade única.
    • src/commons/constants
  • src/routers
    • Rotas
      • Criar um arquivo index.js para importar e exportar todas as rotas.
      
                                import { react-router } from 'react-router';
                                import { BrowserRouter, Route, Switch } from 'react-router-dom';
                                import { Home, Login, Register, Profile, NotFound } from './containers';

                                export const Routes = () => (
                                    <BrowserRouter>
                                        <Switch>
                                            <Route exact path="/" component={Home} />
                                            <Route exact path="/login" component={Login} />
                                            <Route exact path="/register" component={Register} />
                                            <Route exact path="/profile" component={Profile} />
                                            <Route component={NotFound} />      
                                        </Switch>
                                    </BrowserRouter>
                                );
      Depois basta importá-las onde for usar.
      Ex: import { Routes } from './routers';

      Outro exemplo: Seria criar e exportar um objeto contendo todas as rotas, depois importar e realizar uma iteração na hora de renderizar.
  • src/resources
    • Chamadas de serviços (recursos)
      • Geralmente se separa, arquivos de APIs e arquivos de serviços.
  • Testes
    • Criar um diretório chamado _tests_ e dentro deste criar os arquivos de testes.
    Nomenclatura padrão: nameFile.test.js
    Isso vale para todos os diretórios listados acima

DICA: Agrupar nos diretórios arquivos que fazem sentido ficarem juntos, que tenham dependência entre si.
