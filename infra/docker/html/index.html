Rodando web page via container com docker!

Repositório da imagem usada para este container: nginx-heviane

  • Container: é um processo que executa de forma isolada da máquina e SO hospedeiros.
    • Não é uma VM, é apenas um processo rodando na máquina. Um processo fechado.
    Os dados do container se perdem quando o container é destruído.
  • volume: é o diretório local compartilhado e mapeado com o container
    • Neste exemplo, é o diretório "./html".
    Assim os dados do projeto ficam seguros e não são perdidos quando o container é destruído.
  • Imagem: é a base para a criação do container.
    • É imutável, isso significa que alterações realizadas no container não são replicadas na imagem.
  • Dockerfile: é a receita para criar uma imagem.
    • Docker vai executar tudo que estiver neste arquivo, passo a passo, para criar a imagem.
  • docker-compose.yml: Basicamente, é um manifesto.
    • Ferramenta para auxiliar na execução de comandos (Todos os comandos necessários para subir um container).
